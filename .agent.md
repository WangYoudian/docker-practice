
# Docker Practice — Agent Guide

这是一个 Docker 练习项目，面向从零到生产的逐阶段学习。每个练习目录包含可直接运行的 Dockerfile / compose.yml / 源码，配合 README 步骤说明。

---

## 仓库概览

| 阶段 | 目录 | 内容 | 前置条件 |
|------|------|------|----------|
| Phase 1 | `basics/` | 基础命令：生命周期、端口、卷、日志、环境变量 | 无 |
| Phase 2 | `dockerfiles/` | Dockerfile 编写：多阶段、缓存、瘦身、ENTRYPOINT/CMD | Phase 1 |
| Phase 3 | `compose/` | 多容器编排：Web+Redis/DB、依赖、多环境、扩展 | Phase 1-2 |
| Phase 4 | `best-practices/` | 生产化：安全、资源、网络、日志、CI/CD | Phase 1-3 |
| 笔记 | `notes/` | 调研笔记：Podman/nerdctl/Lima 等替代品对比 | - |

## 目录与命名规范

- **阶段目录**：`Phase{番号}-{主题}`，但本 repo 使用 `{主题}/` 形式（`basics/`、`dockerfiles/`、`compose/`、`best-practices/`）
- **练习目录**：`{序号}-{标题英文}`，如 `01-hello-world/`、`02-container-lifecycle/`
- **文件名**：
  - 练习说明：`README.md`
  - Dockerfile：`Dockerfile`，变体用 `Dockerfile.{variant}`（如 `Dockerfile.good`、`Dockerfile.slim`）
  - Compose：`compose.yml`，变体用 `compose.{variant}.yml`（如 `compose.health.yml`、`compose.prod.yml`）
  - 源码：`{语言}.{后缀}`（如 `app.py`、`app.js`、`main.go`）
  - 依赖文件：`requirements.txt`、`package.json`

---

## 练习文件规范

每个练习应包含：

1. **README.md** — 目标说明 + 分步操作 + 清理命令 + 检查清单 + 思考题
2. **Dockerfile 或 compose.yml** — 可直接构建/运行
3. **源码文件** — 实现练习中的应用逻辑（Python/Node/Go 等）
4. **辅助文件** — 视需要（`.dockerignore`、`nginx.conf`、`init.sql` 等）

README 内容结构：

```markdown
# 练习 {N}：{标题}

**目标**：一句话说明

**涉及**：关键指令或概念

---

## Step 1：...

    # 命令用缩进代码块

## 清理

    # 清理命令

## 检查清单

- [ ] 每项关键技能

## 思考题

1. 启发性的问题，促使用户思考
```

## 开发与构建指引

### 本地运行

```bash
# Phase 1（文档练习，无需构建）

# Phase 2
cd dockerfiles/01-first-dockerfile
docker build -t hello-app .
docker run -d -p 8000:8000 hello-app

# Phase 3
cd compose/01-web-redis
docker compose up -d

# Phase 4
cd best-practices/02-resource-limits
docker build -t stress-test .
docker run --rm --cpus=0.5 stress-test
```

### 代码风格

- Dockerfile 遵循 [hadolint](https://github.com/hadolint/hadolint) 规则
- 基础镜像固定版本（如 `python:3.11-slim` 而非 `python:latest`）
- compose.yml 使用版本无关格式（v2 以上，不写 `version:` 字段）
- 源码优先用 Python 标准库，减少外部依赖

## Agent 行为准则

1. **阅读上下文**：先查看 `.agent.md` 和目录结构，理解仓库当前状态后再行动
2. **保持练习结构一致**：新练习遵循已有练习的命名和格式
3. **渐进式设计**：练习之间有关联依赖，不要跳过前置阶段
4. **端口不冲突**：使用约定端口段，避免端口冲突
5. **分段提交**：git 提交按逻辑分组，commit message 以 `phase{番号}({目录}):` 为前缀
6. **可运行性**：每个练习目录提交前需确保 docker build / compose up 可执行
7. **中文说明**：README、练习说明、思考题使用中文
8. **源码与配置**：Dockerfile、源码文件名、代码内标识符使用英文
9. **不破坏已有历史**：不修改已提交的练习文件，只新增或在前置练习的基础上扩展
10. **附带思考题**：每个练习至少 3 个思考题，引导用户深入理解原理
